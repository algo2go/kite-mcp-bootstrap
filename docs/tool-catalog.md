# Kite MCP Tool Catalog

Comprehensive index of every MCP tool exposed by this server, grouped by
category. Generated by reading the registration block in
[`mcp/mcp.go`](../mcp/mcp.go) (func `GetAllTools`) and the per-tool
`Tool()` definitions.

- **Total registered tools:** 111
- **Hosted (ENABLE_TRADING=false, Path 2) exposes:** 93
- **Trading-gated tools:** 18 (stripped on hosted, on in local when
  `ENABLE_TRADING=true`)
- **Billing-gated tools:** 1 (`options_greeks` — TierPremium). Everything
  else is TierFree or TierPro. See [`kc/billing/tiers.go`](../kc/billing/tiers.go).

## Legend

| Icon | Meaning |
|------|---------|
| 📊 | Read-only (`readOnlyHint=true`) |
| ⚙️ | Mutation / writes state |
| 🛑 | Gated off on hosted (`ENABLE_TRADING=false`); available on self-host with `ENABLE_TRADING=true` |
| 🔒 | Admin-only (gated by `IsAdmin` role check, not billing) |
| 💎 | Billing-gated Premium (`options_greeks` only) |
| 🪙 | Billing-gated Pro (write-side tools: orders, GTT, alerts, trailing stops, MF, Telegram, native alerts) |

Tools without an icon are TierFree and read-only or non-destructive.

## Hosted-vs-local gating summary

| Deployment | Env flag | Exposes |
|---|---|---|
| Hosted (Fly.io, Path 2, SEBI-safe) | `ENABLE_TRADING=false` (default) | 93 tools — all read-only + sandbox + admin; no order placement |
| Self-host (personal use) | `ENABLE_TRADING=true` | All 111 tools including the 18 write-side (🛑) ones |

place_order   modify_order   cancel_order   convert_position
close_position   close_all_positions

## Billing tiers

Tier mapping lives in [`kc/billing/tiers.go`](../kc/billing/tiers.go) as the
`toolTiers` map. Current assignments:

- **TierFree** — all read-only data, analytics, paper trading, watchlists,
  admin tools (admin gated by role, not billing), observability, research
  copilot.
- **TierPro** — write-side tools (order lifecycle, GTT, alerts, trailing
  stops, MF orders, Telegram setup, native alerts). Gated here because they
  carry real financial risk; remains the dividing line if billing is ever
  re-activated.
- **TierPremium** — `options_greeks` only. Advanced derivatives analytics.
- **TierSoloPro** — same tool access as Pro (distinguished only by
  `max_users=1`, not feature gates).

Billing infrastructure is dormant today: analytics that previously lived
under Pro/Premium have been moved to Free per Path 1 commitment (stay
free forever under SEBI MCP framework).

## Telegram

Telegram is a **notification channel**, not an MCP tool set. The
`setup_telegram` tool is the only Telegram-facing MCP surface and only
registers the user's chat ID. Scheduled morning/EOD briefings and trading
commands (`/buy`, `/sell`, `/quick`, `/setalert`) are handled by the
`kc/telegram` package directly, not via MCP. See
[`docs/kite-token-refresh.md`](kite-token-refresh.md) §
Telegram for details.

## Annotations on every tool

Every tool registers MCP annotations via `mcp.WithTitleAnnotation`,
`WithReadOnlyHintAnnotation`, `WithDestructiveHintAnnotation`,
`WithIdempotentHintAnnotation`, `WithOpenWorldHintAnnotation`. These feed
the MCP client UI (e.g., Claude Desktop's confirmation dialogs + tool
icons). Clients without annotation support fall back to invocation
without confirmation prompts.

## MCP prompts (not tools)

Server also registers three MCP prompts — `morning_brief`,
`trade_check`, `eod_review` — via `mcp.RegisterPrompts`. These are
server-side prompt templates, not tools; see [`mcp/prompts.go`](../mcp/prompts.go).

## App widgets (MCP Apps)

A subset of tools attach `_meta["ui/resourceUri"]` entries that point to
inline widget pages (portfolio, activity, orders, alerts). See
[`mcp/ext_apps.go`](../mcp/ext_apps.go) and
[`kc/templates/`](../kc/templates/) for widget resources and
`AppBridge` plumbing.
